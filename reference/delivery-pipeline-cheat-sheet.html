Delivery pipeline

Qué debe ocurrir, qué evidencia mirar y qué hacer cuando falla.

Flujo

issue → shape → implement → build once → acceptance → preview → promote → verify

Invariantes

Reglas que no negociamos

`main` releasable

Rojo detiene integración normal.

Mismos bytes

Testear, desplegar y recuperar sin rebuild.

Cambios pequeños

Feedback rápido, diagnóstico y reversión simples.

Diagnóstico

La falla indica dónde aprender

Gate Pregunta Respuesta inicial
Local ¿El cambio es coherente? Corregir antes de push.
Commit stage ¿Vale la pena seguir? Restaurar verde inmediatamente.
Acceptance ¿El artefacto se comporta bien? Conservar diagnósticos; no promover.
Preview smoke ¿Corre en infraestructura real? Investigar deploy/configuración.
Production smoke ¿La promoción quedó saludable? Promover known-good y diagnosticar.

Wayfinding

Comandos esenciales

Feedback completo local pnpm verify
Ver corridas gh run list --workflow delivery.yml
Seguir una corrida gh run watch <run-id> --exit-status
Preview — próximo vercel deploy --prebuilt
Promote — próximo vercel promote <url> --yes
Recovery — próximo vercel promote <known-good-url> --yes

DORA

Medir el sistema, no premiar números

Throughput

Lead time, deployment frequency y failed deployment recovery time.

Instability

Change fail rate y deployment rework rate.
